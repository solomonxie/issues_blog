# VIM加速


## 找出影响速度的环节

```sh
$ vim --startuptime /tmp/vim.log ~/.vimrc +qall && cat /tmp/vim.log |sort -nrk 2
```
然后就能看到各个环节加载时间,小数点前面是s秒，小数点后面是毫秒。
其中第一列是时间点，第二列是时长，我们主要关注第二列。
一般标准： ”200ms 以下感觉是很好的，超过 500ms 会觉的有点卡，如果超过 1s 就会觉得非常难受了“
