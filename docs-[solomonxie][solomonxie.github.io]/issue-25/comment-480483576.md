## 为什么要关闭Swap？

Swap的缺点是：
- 如果同时编辑两个项目的同名文件，那么也会提示conflict冲突。而解决这个问题并不容易
- 如果已经保存了Undo历史，那么Swap就没有什么必要性了。如果是在Git版本控制下，就更没存在意义了。

如果要取消Swap，可以考虑这些更好的方案：
- 设置Undo
- 设置每次Buffer获得焦点时自动刷新文档内容 （如果同时在多处编辑同一个文件，很方便）
